| 0.1           | Initial version: Core thinking agent with XML-based tool execution.                                                                 | Model can execute tools defined with XML tags within its thought process.                                                                                      | None (Baseline)                                                                                                                                                 |
| 0.2           | Implemented foldable tool descriptions.                                                                                             | Enhances clarity and reduces initial prompt size by hiding tool details by default. Details are shown on demand (e.g., via a `toolbox` or `toolhelp` command). | None observed.                                                                                                                                                  |
| 0.3           | Integrated JSON schema for folded tool calls.                                                                                       | Enabled JSON-based tool calls, aiming for broader compatibility by adopting a more standard format over XML.                                                   | Model occasionally struggles to differentiate between using JSON tool calls and predefined XML tags.                                                            |
| 0.4           | Replaced all tool invocation mechanisms with JSON tool calls.                                                                       | Aimed for maximum compatibility with standard tool-calling conventions.                                                                                        | Resulted in the poorest performance, including degraded response quality and increased instances of non-productive loops.                                       |
| 0.5           | Embedded comprehensive report-writing guidelines within a foldable tool (building on 0.2).                                          | Improved the quality and structure of final reports.                                                                                                           | Led to reduced performance in multi-turn conversations. The model also showed less inclination to use tools in later turns after the guidelines were displayed. |
| 0.6           | Appended predefined text after tool responses, simulating model-generated continuations (e.g., "Tool call has been executed. Now"). | Effectively resolved many existing dead-loop scenarios by encouraging the model to continue its thought process.                                               | None observed.                                                                                                                                                  |
